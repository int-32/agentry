# EARS / BDD / TDD 文档体系巡检

这是周期性巡检任务。每次巡检都执行，不要因为已有执行记录就跳过。

## 目标

定期检查本项目的 EARS / BDD / TDD 文档体系是否建立完整、位置是否正确、内容是否仍然准确。发现文档缺漏、路径过时、引用不一致时，可以直接修正文档；如果涉及产品边界或实现决策，不要猜，记录为待确认问题。

## 巡检范围

重点检查：

- `docs/specs/README.md`
- `docs/specs/AUTHORING.md`
- `docs/specs/EARS.md`
- `docs/specs/BDD.md`
- `docs/specs/TDD.md`
- `docs/specs/features/*.md`
- 各实现模块下的 `EARS.md`、`BDD.md`、`TDD.md`

## 检查要求

1. 确认项目采用“全局文档 + 模块本地文档”的混合结构。
2. 全局 `docs/specs/` 负责规则、索引、模板和跨模块 feature。
3. 能归属到具体实现模块的能力，模块下必须有三件套：
   - `EARS.md`
   - `BDD.md`
   - `TDD.md`
4. 检查文档中的代码路径是否仍然存在。
5. 检查文档中的测试文件是否仍然存在。
6. 检查 EARS / BDD / TDD ID 是否互相引用一致。
7. 检查新增的重要模块是否缺少对应文档。
8. 不要修改业务代码，只处理文档问题。
